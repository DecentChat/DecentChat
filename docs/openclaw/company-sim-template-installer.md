# Company-sim template installer (operator guide)

This guide documents the first shippable template-first install path for company-sim.

## What this flow does

Given a template (for now: `software-studio`) plus a small set of answers, the installer will:

- materialize `company-sims/<company-id>/company.yaml`
- copy template docs (`COMPANY.md`, `ORG.md`, `WORKFLOWS.md`, role files)
- scaffold per-agent workspaces under `.company-sim/workspaces/<company-id>/...`
- merge `agents.list` + `bindings` into your OpenClaw config
- create/merge `channels.decentchat.accounts.*` entries for template employees
- set channel-level `companySimBootstrap` defaults so runtime bootstrap is wired automatically

## Preconditions

- You run from the monorepo (`/Users/claw/Projects/decent-chat`)
- You have a valid OpenClaw config file (JSON or YAML)
- Any *new* DecentChat account generated by installer must receive a valid `seedPhrase` before runtime bootstrap can succeed

## 1) List available templates

From `decent-openclaw/`:

```bash
cd /Users/claw/Projects/decent-chat/decent-openclaw
bun -e "import { listCompanySimTemplates } from './src/company-sim/template-registry.ts'; console.log(listCompanySimTemplates().map(t => ({ id: t.id, label: t.label, description: t.description })));"
```

## 2) Install template into config

Use a small script that:

1. loads your current OpenClaw config
2. loads template metadata (`getCompanySimTemplate`)
3. runs `installCompanyTemplate`
4. writes the merged config back

Example (JSON config, adjust paths as needed):

```bash
cd /Users/claw/Projects/decent-chat/decent-openclaw
bun -e "
import { readFileSync, writeFileSync } from 'node:fs';
import { resolve } from 'node:path';
import { getCompanySimTemplate } from './src/company-sim/template-registry.ts';
import { installCompanyTemplate } from './src/company-sim/template-installer.ts';

const configPath = resolve(process.cwd(), '../openclaw.config.json');
const cfg = JSON.parse(readFileSync(configPath, 'utf8'));

const template = getCompanySimTemplate('software-studio');
const result = installCompanyTemplate({
  template,
  config: cfg,
  answers: {
    companyName: 'Acme Platform',
    workspaceName: 'Acme HQ',
    managerAlias: 'Mira PM',
    backendAlias: 'Devon API',
    qaAlias: 'Iva QA',
  },
  workspaceRootDir: resolve(process.cwd(), '..'),
});

writeFileSync(configPath, JSON.stringify(result.config, null, 2), 'utf8');
console.log(JSON.stringify(result.summary, null, 2));
"
```

## 3) Fill seed phrases for newly created accounts

`result.summary.createdAccountIds` tells you which account ids were created by installer.

For each new account under `channels.decentchat.accounts.<accountId>`:

- set a valid 12-word `seedPhrase`
- keep generated `companySim` metadata as-is
- keep generated `dataDir` unless you intentionally relocate it

Without seed phrases, startup bootstrap will fail with a missing/invalid account seed error.

## 4) Start OpenClaw and verify bootstrap

On startup, each account with `companySimBootstrap.enabled=true` and `mode=runtime` will run workspace bootstrap from the generated manifest.

Quick checks:

- logs include `company bootstrap ready`
- generated workspace exists for all company accounts
- required channels and bot members are present

## 5) Idempotency expectations

Re-running installer with the same template + answers should be safe:

- existing managed agents/bindings are updated in place (not duplicated)
- account entries are merged (not replaced wholesale)
- scaffold + manifest paths stay stable for the same `companyId`

## Current limitations

- No dedicated top-level operator CLI command yet (installer is used through TypeScript API)
- Advanced template export/import path is separate (power-user flow)
