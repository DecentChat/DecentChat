## ConnectionRetryProgress Review

**Status: PASS — well-implemented.**

- Pure state machine in `ConnectionRetryProgress.ts` — testable, no DOM coupling.
- Backoff delays: 0ms, 800ms, 2s, 5s, 10s — sensible progressive backoff.
- Attempt 0 is silent (no flicker on fast connections). Attempts 1-4 show "Connecting to network… attempt N/5".
- On exhaustion: toast "Offline mode — messages will sync when connected" (info, 6s).
- 9 unit tests all pass covering: happy path, retry-then-success, full exhaustion, edge cases.
- Loading screen has `role="status"` + `aria-live="polite"` — screen readers announce changes.
- Spinner has `aria-hidden="true"` — correct for decorative element.

## Key friction — REMAINING (all LOW severity)

### [P1] Channel header button overload (V2)
**Severity: LOW**
**Status: Partially mitigated**

Non-DM channels show 6 icon buttons (🎧🔌📱🔍🔗⚙️). At ≤390px, QR/Connect/Invite buttons are hidden via `display: none` — removes access entirely.

**Fix:** Consolidate into 3 primary visible + overflow "⋯" menu component.
**Files:** `ChannelHeader.svelte` + new `OverflowMenu.svelte` + CSS
**Owner:** Codex Engineer

### [P2] "AI" sidebar button is cryptic
**Severity: LOW**

"AI" button with tooltip "Install AI team template" — unclear purpose to new users.
**Owner:** Product decision needed.

### [P3] Modal accessibility
**Severity: LOW**

`ModalImpl.svelte` uses `<div class="modal-overlay">` instead of native `<dialog>`. No `aria-hidden` on background content when modal is open. Screen reader users can navigate to background content while modal is active.

**Fix options:**
- Add `aria-hidden="true"` to `#app` content when modal is open
- Or migrate to native `<dialog>` with `showModal()` for automatic inert background
**Owner:** Design decision + Codex Engineer

## What's good

- **Copy quality is excellent throughout.** Every piece of user-facing text is clear, calm, and purposeful. "Offline mode — messages will sync when connected" is a textbook offline state message.
- **Semantic HTML is correct.** Comparison table uses `<table>`, `<thead>`, `<th scope>`, `<tbody>`. Proper ARIA attributes on loading screen.
- **Connection retry is invisible on happy path.** First attempt shows no UI change — users with fast connections never see retry messaging.
- **Error states are differentiated.** Storage issues get specific guidance ("Close other tabs") vs generic errors ("Try reload first").
- **Single-tab enforcement** is user-friendly — clear explanation + "Reload this tab" CTA.
- **Footer peer ID** has proper click-to-copy with visual hover affordance.
- **Problem banner** creates effective competitive context without feeling aggressive.
- **Tech pills section** builds credibility with specific, real technology names.
- **Build is fast** — 2.08-2.14s consistently.
