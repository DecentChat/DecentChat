# DecentChat UX Review — 2026-04-04
**Reviewer:** Iris UX
**Scope:** Landing page, onboarding flow, main app (empty state), settings panel

---

## UX verdict: PARTIAL

The product has strong fundamentals — clear value proposition, good information architecture, comprehensive feature set. However, there are accessibility failures, loading state gaps, and some information density issues that hurt first-run experience.

---

## Screens reviewed

1. **Landing page** (http://localhost:5173/) — hero, feature grid, comparison table, tech pills, donation section, footer CTA
2. **Loading/splash screen** — "Opening your secure message history..."
3. **Main app — empty state** (#general channel with no messages)
4. **Settings panel** — identity, appearance, notifications, privacy, media, advanced, danger zone
5. **Mobile responsive behavior** (CSS review, not live device)

---

## Key friction

### P0 — Loading state has no timeout feedback
When the signaling server is unreachable (or slow), the user sees "Opening your secure message history..." for ~18 seconds with zero feedback. No progress bar, no retry count, no "working offline" fallback message. The retry loop ([0, 800, 2000, 5000, 10000]ms in main.ts:938-965) is invisible to the user.

**Impact:** New users hitting this on first launch will think the app is broken and leave. This is the #1 onboarding killer.

**Recommendation:** Show retry progress ("Connecting to network... attempt 3/5") and when falling through to offline mode, show a brief toast: "Offline mode — messages will sync when connected."

### P1 — Channel header button overload
The #general channel header has **7 emoji icon buttons** in a row: 👥 🎧 🔌 📱 🔍 🔗 ⚙️. For new users, this is visually dense and none of these have text labels. The icons don't all have universal meaning (what does 🔌 mean to a first-time user?).

**Recommendation:** Group infrequently-used actions (Connect to peer, QR Code) behind a "⋯" overflow menu. Keep primary actions (Search, Members, Settings) visible. Consider text labels on hover or a reduced set of 3-4 buttons.

### P1 — Mobile comparison table loses context
At ≤768px, the middle column ("Discord / WhatsApp / Telegram") is hidden via `display: none`. Users on mobile see Feature ↔ DecentChat without knowing what they're comparing against. The competitive context — the whole point of the section — is lost.

**Recommendation:** Either stack rows vertically with both columns labeled, or use a "tap to compare" interaction that shows both sides.

## Visual issues

### Fixed: WCAG AA contrast failure on landing page badges/links
- `.lp-hero-badge` and `.lp-inline-link` used `color: #5b4fd1` on dark background (`#0f0f23`).
- Contrast ratio: **2.58:1** — fails WCAG AA (requires 4.5:1 for normal text).
- **Fix applied:** Changed to `var(--primary-light)` (`#a29bfe`), which provides **6.41:1** contrast ratio, passing WCAG AA and AAA.

### Minor: Comparison table uses divs instead of semantic `<table>`
The comparison data is laid out with CSS Grid on `<div>` elements. Screen readers won't understand it as tabular data. Consider adding `role="table"`, `role="row"`, `role="columnheader"`, `role="cell"` ARIA roles.

### Minor: Hardcoded color values in comparison rows
`.lp-compare-row .bad` uses hardcoded `#e74c3c`, `.mid` uses `#d4870e`, `.good` uses `#00b894`. These should use CSS variables for consistency with the design system (they already match `--error` and `--success`/`--online` but aren't referencing them).

## Suggested improvements

1. **Loading screen: Add progress feedback and offline fallback messaging** (P0)
2. **Header: Reduce visible button count, add overflow menu** (P1)
3. **Mobile comparison: Preserve competitive context** (P1)
4. **Comparison table: Add ARIA table roles** (P2)
5. **Design system: Replace hardcoded color values with CSS variables** (P2)
6. **Empty state: The current empty state is actually good** — clear next steps, two prominent CTAs (invite link + QR), helpful keyboard shortcut hints. No changes needed.
7. **Settings panel: Well organized** — clear section hierarchy, sensible defaults. No changes needed.
8. **Landing page copy: Strong** — clear value prop, good competitive positioning, honest tone. The "no signup" messaging is consistent throughout.
