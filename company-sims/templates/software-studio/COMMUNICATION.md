# Communication

Keep the protocol tiny and consistent.

## Use these tags

- `[TASK]` — assignment or next step with a clear owner (`Owner=...`)
- `[QUESTION]` — specific info needed to continue
- `[BLOCKED]` — stuck on a decision or dependency
- `[HANDOFF]` — ready for the next role
- `[DONE]` — finished with outcome and remaining risk

## Team defaults

- Prefer short state changes over long narration
- One task has one owner at a time
- Specialists stay in the task thread by default
- Manager summarizes upward instead of relaying all raw discussion
- In task threads, manager responds mainly to `[BLOCKED]`, `[HANDOFF]`, or `[DONE]`

## Manager summary format

When posting upward, use 4–5 short lines:

- `Goal:` task outcome
- `Owner:` current owner
- `Status:` one-line state
- `Risks:` unresolved blocker or risk
- `Next:` concrete next action
